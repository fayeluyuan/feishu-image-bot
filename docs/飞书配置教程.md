# 飞书机器人配置教程

本文档手把手教你如何在飞书开放平台上创建应用、配置机器人、并将本服务接入飞书。

---

## 第一步：创建飞书应用

1. 访问 [飞书开放平台](https://open.feishu.cn/app)
2. 登录后点击 **"创建企业自建应用"**
3. 填写应用名称（如"AI图片助手"），选择应用类型为 **"企业自建应用"**
4. 点击"确定创建"

---

## 第二步：获取 App ID 和 App Secret

1. 进入应用详情页，点击左侧 **"凭证与基础信息"**
2. 复制以下两个值，填入本项目的 `.env` 文件：
   - **App ID** → `FEISHU_APP_ID`
   - **App Secret** → `FEISHU_APP_SECRET`

---

## 第三步：开启机器人能力

1. 点击左侧菜单 **"机器人"**
2. 打开 **"启用机器人"** 开关
3. 配置机器人信息：
   - 机器人名称：`AI图片助手`
   - 机器人描述：`发送图片描述，我帮你生成图片并上传到云盘`
   - 消息卡片/菜单：可选，保持默认即可
4. 点击"保存"

---

## 第四步：配置权限

机器人需要以下权限才能正常工作：

1. 点击左侧 **"权限管理"**
2. 在搜索框中依次添加以下权限：
   - `im:chat:readonly` — 获取群组信息
   - `im:message` — 收发消息
   - `im:message.group_msg` — 发送群消息
   - `im:message:send_as_bot` — 以机器人身份发送消息
   - `drive:drive` — 访问云盘
   - `drive:file:read` — 读取云盘文件
   - `drive:file:write` — 写入云盘文件
3. 每添加一个权限后点击 **"批量申请"**
4. 企业管理员会收到审批通知，需要**管理员审批通过**

---

## 第五步：配置事件订阅

1. 点击左侧 **"事件与回调"** → **"事件订阅"**
2. 打开 **"启用事件订阅"** 开关
3. 在 **"请求网址配置"** 中填入你的服务地址：
   - 本地测试：`https://你的ngrok地址.ngrok-free.app/webhook`
   - 云服务器：`http://你的服务器IP:5000/webhook`
4. 在 **"添加事件"** 中搜索并添加：
   - `接收消息`（事件类型：`im.message.receive_v1`）
5. 点击"保存"

> 首次保存时，飞书会发送一个 challenge 请求到你的服务器进行验证。确保你的服务已经启动并且能正常访问。

---

## 第六步：本地测试（使用 ngrok）

如果你还没有云服务器，可以用 ngrok 做内网穿透进行本地测试：

1. 安装 ngrok：
   ```bash
   # Ubuntu/WSL
   curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc >/dev/null
   echo "deb https://ngrok-agent.s3.amazonaws.com buster main" | sudo tee /etc/apt/sources.list.d/ngrok.list
   sudo apt update && sudo apt install ngrok
   ```

2. 注册 ngrok 账号，获取 authtoken：
   ```bash
   ngrok config add-authtoken 你的token
   ```

3. 启动 ngrok 穿透：
   ```bash
   ngrok http 5000
   ```

4. 复制 ngrok 提供的 HTTPS 地址（如 `https://abcd1234.ngrok-free.app`），加上 `/webhook` 填入飞书事件订阅。

---

## 第七步：发布应用

1. 点击左侧 **"版本管理与发布"**
2. 点击 **"创建版本"**
3. 填写版本信息：
   - 版本号：`1.0.0`
   - 更新说明：`初始版本，支持图片生成`
4. 点击"保存"
5. 点击 **"申请发布"**
6. 等待企业管理员审批

审批通过后，机器人就会出现在你的飞书企业/团队中，可以 @它开始使用了。

---

## 常见问题

### Q: 事件订阅验证失败（challenge 失败）

确保你的服务已经启动，并且 `/webhook` 路径可以访问。飞书会发送一个 POST 请求，包含 `challenge` 字段，你的服务需要原样返回这个值。

### Q: 机器人收不到消息

检查以下事项：
- 事件订阅中是否添加了 `im.message.receive_v1` 事件
- 权限管理中是否申请了消息相关权限
- 应用是否已经发布并通过审批
- 机器人是否已经被添加到群聊中

### Q: 图片生成成功但没上传到云盘

检查权限管理中是否申请了 `drive:drive`、`drive:file:read`、`drive:file:write` 权限，并且管理员已审批通过。

### Q: 本地测试正常，部署到云服务器后飞书收不到消息

- 检查云服务器防火墙是否放行了 5000 端口
- 检查安全组规则（腾讯云需要在控制台配置安全组）
- 确保使用公网 IP 或域名访问
